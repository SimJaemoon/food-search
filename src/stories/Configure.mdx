## **작성 규칙**


* 컴포넌트를 **뼈대**(Presentational) 와 **동작**(Container) 으로 구분하는 설계 방식을 채택함  
<a href="https://medium.com/@dan_abramov/smart-and-dumb-components-7ca2f9a7c7d0" target="_blank">Dan Abramov(2015), Presentational and Container Components</a>
* Presentational 컴포넌트를 __심리적 크기__ 에 따라 구분하기 위해 **Atomic Design** 을 참고해 분류함  
<a href="https://atomicdesign.bradfrost.com/chapter-2/" target="_blank">Brad Frost, Atomic Design Methodology</a>

<br/>

#### &nbsp;&nbsp; **Atomic Design**  
##### &nbsp;&nbsp; _대상 : Presentational Component (UI)_
* __atom__(원자) : 최소 표현 단위  
* __molecule__(분자) : 1개 이상 atom 포함 ( atom ≥ 1 )  
* __organism__(유기체) : 1개 이상 molecule 포함 ( molecule ≥ 1 ＋ atom ≥ 0 )

<br/>
<br/>

### 구현하며 느낀 점
* * *
* 컴포넌트를 성격(UI, Feature)에 따라 구분하면, Presentational(UI) 컴포넌트 작성시 자연스럽게 **다양한 사용 맥락**에 대응할 수 있도록 prop 을 설계하게 됨
  * 즉, 컴포넌트의 **재사용성**을 무의식적으로 높이는 결과를 가져옴
* 하지만, **간단한 Feature 도 항상 Container 컴포넌트로 분리해야 하는 점이 비효율**적으로 느껴짐
  * 실제로, 작업 중 UI, Feature 관련 code 가 하나의 컴포넌트에 공존하는 경우 발생함
    * molecules ( CategoryDetailCarousel, ProductListStand )
    * organism ( ProductCardGroup )
    * Container & modal page
